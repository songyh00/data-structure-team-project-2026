# Campus Weighted Graph Algorithm Comparison

데이터구조 수업의 **가중치 그래프** 단원을 기반으로 구현한 C언어 프로젝트입니다.

캠퍼스 건물 이동 경로를 가중치 그래프로 구성하고,  
MST 알고리즘과 최단거리 알고리즘의 결과를 비교합니다.

---

## 프로젝트 목표

- 캠퍼스 건물 이동 경로를 가중치 그래프로 표현
- MST 알고리즘 3개 비교
- 최단거리 알고리즘 3개 비교
- 사용자가 입력한 출발지/도착지에 따른 최단경로 출력
- 동일한 그래프에서 여러 알고리즘의 결과가 일치하는지 확인

---

## 그래프 구성

### 정점(Vertex)

| 번호 | 건물 |
|---:|---|
| 0 | 정문 |
| 1 | 성결관 |
| 2 | 학생회관 |
| 3 | 중생관 |
| 4 | 기념관 |
| 5 | 재림관 |
| 6 | 학술정보관 |

### 간선(Edge)

가중치는 건물 사이의 예상 이동 시간입니다.

```text
정문 -- 성결관 : 3분
정문 -- 기념관 : 7분
정문 -- 재림관 : 5분
성결관 -- 학생회관 : 1분
성결관 -- 중생관 : 2분
성결관 -- 기념관 : 5분
성결관 -- 재림관 : 4분
학생회관 -- 중생관 : 2분
중생관 -- 기념관 : 3분
기념관 -- 재림관 : 3분
재림관 -- 학술정보관 : 1분
```

---

## 사용 알고리즘

### MST 알고리즘

- Kruskal
- Prim
- Boruvka

MST는 특정 출발지와 도착지를 찾는 알고리즘이 아니라,  
전체 정점을 최소 비용으로 연결하는 알고리즘입니다.

### 최단거리 알고리즘

- Dijkstra
- Floyd-Warshall
- Bellman-Ford

사용자가 출발지와 도착지를 입력하면,  
각 알고리즘이 계산한 최단거리와 최단경로를 출력합니다.

---

## MST 실행 결과

세 알고리즘 모두 MST의 총 비용은 `13분`으로 계산되었습니다.

| 알고리즘 | 총 비용 |
|---|---:|
| Kruskal | 13분 |
| Prim | 13분 |
| Boruvka | 13분 |

예시 MST 선택 간선은 다음과 같습니다.

```text
성결관 -- 학생회관 : 1분
재림관 -- 학술정보관 : 1분
성결관 -- 중생관 : 2분
정문 -- 성결관 : 3분
중생관 -- 기념관 : 3분
기념관 -- 재림관 : 3분
```

알고리즘마다 간선을 선택하는 순서는 다를 수 있지만,  
같은 그래프에 대한 MST의 총 비용은 동일하게 나와야 합니다.

---

## 최단거리 실행 예시

입력:

```text
출발지 번호 입력: 0
도착지 번호 입력: 6
```

결과:

```text
출발지: 정문
도착지: 학술정보관

[Dijkstra]
최단거리: 6분
최단경로: 정문 -> 재림관 -> 학술정보관

[Floyd-Warshall]
최단거리: 6분
최단경로: 정문 -> 재림관 -> 학술정보관

[Bellman-Ford]
최단거리: 6분
최단경로: 정문 -> 재림관 -> 학술정보관
```

---

## 메뉴 기능

프로그램은 MST 결과를 먼저 출력한 뒤,  
사용자가 원하는 출발지와 도착지를 선택하여 최단경로를 반복해서 검색할 수 있습니다.

```text
1. 최단경로 검색
0. 종료
```

---

## 정리

이 프로젝트에서는 동일한 캠퍼스 가중치 그래프에 여러 알고리즘을 적용하여 결과를 비교했습니다.

- MST 알고리즘은 총 비용이 모두 동일하게 나오는지 확인
- 최단거리 알고리즘은 최단거리와 최단경로가 동일하게 나오는지 확인
- 실제 캠퍼스 이동 상황을 그래프로 표현하여 알고리즘 적용 과정을 확인
